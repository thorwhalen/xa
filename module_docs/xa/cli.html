xa.cli

Command-line interface for xa.

Exposes subcommands via argh: list, info, history. Later phases add spawn, resume, kill, serve and others.

Entry point: xa (see [project.scripts] in pyproject.toml).

List Claude Code sessions across configured hosts, newest first.

Parameters:

  • limit – Maximum rows to show (0 = unlimited).

  • include_forks – Include sessions spawned with claude --resume.

xa.cli.serve_cmd(host: str = '127.0.0.1', port: int = 8010, mount: str = '', username: str | None = None, password: str | None = None, password_env: str | None = None, captcha: bool = False, webui: bool = True, i_know_its_insecure: bool = False) → None[source]

Run the xa HTTP service (requires the xa[service] extra).

Secure by default. Refuses to bind to a non-loopback interface without HTTP Basic credentials. Override this guardrail with --i-know-its-insecure only in locked-down deployments (e.g. behind an mTLS-terminating proxy).

Parameters:

  • host – Bind host. 127.0.0.1 by default — only this machine.

  • port – Bind port.

  • mount – Optional mount prefix (e.g. /api/xa).

  • username – HTTP Basic username.

  • password – HTTP Basic password. Prefer --password-env.

  • password_env – Name of env var to read the password from (recommended).

  • captcha – Enable captcha-gated deletes. Strongly recommended for any non-loopback deployment.

  • webui – Serve the bundled web UI at the mount root (default on). Pass --no-webui to run API-only.

  • i_know_its_insecure – Skip the public-bind-without-auth safety check.
